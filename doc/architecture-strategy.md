# SURVIVAL VOXEL — アーキテクチャと現在のプロジェクト状況

本ドキュメントは、これまでのリファクタリングおよび機能拡張で実現された最新のプロジェクト構造とアーキテクチャ、および現在の達成状況をまとめたものです。

---

## 1. 現在の達成状況と追加実装機能

当初計画していた `index.html` へのベタ書きからのコンポーネント・モジュール化（Phase 1 〜 Phase 5）は**完全に完了**し、その上でゲーム性・パフォーマンスを向上させるための多くの高度なシステムが実装・統合されています。

### 完了した主要なアップデート

1. **完全なモジュール分割とViteへの移行**
   - Three.js の npm パッケージ化と ES Modules(ESM) によるコンポーネント分割。
2. **パフォーマンス・描画の最適化**
   - **TextureAtlas (テクスチャアトラス)**: 全ブロックのテクスチャを1枚に結合・動的生成し、ドローコールを極限まで削減。
   - **ChunkWorkerPool (並列チャンク生成)**: `TerrainGenerator` のような重い地形・ノイズ生成処理を Web Worker プールでマルチスレッド化。
   - **Uint8Array (データ構造の最適化)**: チャンクのブロックデータを文字列の `Map` から 64KB の1次元配列(`Uint8Array`)に変更し、メモリ効率とアクセス速度を劇的に向上。
   - **同期再構築によるラグなし描画**: メインスレッドでブロックの設置・破壊時に即座にメッシュを再生成する高速ローカル処理 (ゼロ遅延描画)。
   - **LOD緩和と遠景拡張**: 余剰パフォーマンスを活かし、フル解像度(LOD 0)のレンダリング距離を延長し、最大描画距離も18から24に変更。
3. **ゲームプレイ・機能の拡張**
   - **ワールドシステム (SaveManager)**: 個別シード値、複数ワールドの切り替え、ブロック変更履歴・インベントリ・ホットバーの永続化。
   - **乱数シード (Seed)**: LCGおよびパーリンノイズにワールドごとの固定シード値を導入し、同一マップの再現性を確立。
   - **UIマネージャ統合**: ポーズ画面、インベントリ、設定画面(マウス感度やFOV)、ヘルス・空腹ゲージ、および「ミニマップ(MapUI)」の追加。
   - **操作性の向上**: 右クリック/左クリックの長押しによる、スムーズなブロックの連続設置・連続破壊。

## 3. 次の課題と将来への方向性

本プロジェクトの基盤構築と最適化は十分に成熟しており、今後は純粋なコンテンツの拡張や、マルチプレイヤーのような上位層の機能追加に焦点を当てることができます。

1. **コンテンツ展開**
   - クラフト機能 (`CraftingSystem.js` の本実装と `recipes.json`)
   - Mob（動物、敵対生物）の追加
   - より複雑なダンジョン生成アルゴリズムの TerrainGenerator への組み込み
2. **パフォーマンス限界への挑戦**
   - Index Buffer のさらなる最適化（Greedy Meshing などの採用）による頂点数の削減
   - 遠景用の軽量なビルボード・あるいは巨大なチャンクメッシュへの自動ベイキング
3. **サーバクライアント型のマルチプレイ**
   - `World.js` と `Player.js` のロジックをNode.js (またはリアルタイムサーバ)側に引き上げ、状態をWebSocket(やWebRTC)で同期する `NetworkManager.js` の追加
